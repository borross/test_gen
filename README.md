# test_gen.py — Генератор итоговых тестов по ИБ

Скрипт собирает итоговый тест из одного или нескольких markdown-файлов с тестами.
Поддерживает три типа вопросов, перемешивание вариантов ответов, фильтрацию по
сложности и типу, весовое распределение вопросов по темам. Автоматически формирует
блок правильных ответов — в том числе развёрнутых текстовых ответов на открытые вопросы.

---

## Требования

- Python 3.8+
- Стандартная библиотека (сторонние пакеты не нужны)

---

## Поддерживаемые форматы входных файлов

| Сценарий | Описание |
|----------|----------|
| **1 файл — 1 тест** | Классический одиночный тест-файл |
| **1 файл — N тестов** | Один большой файл, тесты разделены `#` заголовками |
| **Несколько файлов** | Любое сочетание сценариев выше |

---

## Типы вопросов

Скрипт поддерживает три типа вопросов, которые определяются маркером перед заголовком:

| Маркер | Тип | Описание |
|--------|-----|----------|
| `> ℹ️ Базовый` | Базовый тестовый | Вопрос с вариантами A/B/C/D, базовый уровень |
| *(без маркера)* | Обычный тестовый | Вопрос с вариантами A/B/C/D, стандартный уровень |
| `> ℹ️ Открытый` | Открытый | Вопрос без вариантов, требует развёрнутого ответа |

---

## Формат markdown-теста

```markdown
# 15 Безопасность приложений (Всего вопросов: 21, ...)

> ℹ️ Базовый
### 1. В чём принципиальная разница между багом и уязвимостью?
A. Это синонимы
B. Уязвимость — это баг с последствиями для CIA
C. Уязвимость возникает только в опенсорсе
D. Уязвимостями называют только сетевые проблемы

---

### 2. Что означает принцип Shift Left в DevSecOps?
A. Переход разработки на Linux
B. Встраивание проверок безопасности как можно раньше в жизненный цикл
C. Передача кода аудиторам только после релиза в прод
D. Расположение элементов IDE в левой части экрана

---

> ℹ️ Открытый
### 3. Что такое SQL-инъекция и в чём её опасность?

---

## ОТВЕТЫ — 15 Безопасность приложений
1-B, 2-B

**Ответы на открытые вопросы:**

3. SQL-инъекция — уязвимость, при которой злоумышленник внедряет произвольные
SQL-команды в пользовательский ввод, что может привести к утечке данных...
```

### Правила разметки

| Элемент | Формат |
|---------|--------|
| Заголовок теста | `# Название` или `## Название` — любой заголовок 1-2 уровня |
| Аннотация в заголовке | `(Всего вопросов: 21, ...)` или `(15 вопросов)` — автоматически убирается из названия |
| Маркер базового вопроса | `> ℹ️ Базовый` на строке перед `### N.` |
| Маркер открытого вопроса | `> ℹ️ Открытый` на строке перед `### N.` |
| Заголовок вопроса | `### N. Текст вопроса` |
| Варианты ответов | `A. текст`, `B. текст`, `C. текст`, `D. текст` |
| Блок буквенных ответов | Строка, содержащая слово «ОТВЕТЫ», затем `1-B, 2-C, ...` |
| Блок открытых ответов | Строка с «открыт» + «ответ», затем `N. Текст ответа...` |

> **Совместимость:** скрипт нормализует кириллические двойники латинских букв в ответах
> (например, кириллическая **С** → латинская **C**), поэтому `5-С` и `5-C` равнозначны.

## Аргументы командной строки

```
python test_gen.py -i FILE [FILE ...] [опции]
```

### Обязательные

| Аргумент | Описание |
|----------|----------|
| `-i`, `--input FILE [FILE ...]` | Один или несколько входных markdown-файлов |

### Параметры генерации

| Аргумент | По умолчанию | Описание |
|----------|-------------|----------|
| `-n`, `--total N` | — | Общее количество вопросов в итоговом тесте. Обязателен без `--info` |
| `-w`, `--weights W [W ...]` | равные веса | Веса для каждого **файла**. Если файл содержит N тестов, его вес делится поровну между ними. Числа или проценты — сумма не обязана равняться 100 |
| `--type TYPE` | `mix` | Тип вопросов (см. таблицу ниже) |
| `--shuffle` | выкл. | Перемешать варианты ответов внутри каждого тестового вопроса. Правильный ответ пересчитывается автоматически |

### Типы вопросов (`--type`)

| Значение | Что включается в тест |
|----------|-----------------------|
| `mix` | Все тестовые вопросы с вариантами ответа (базовые + обычные) — **по умолчанию** |
| `basic` | Только базовые тестовые вопросы (`> ℹ️ Базовый`) |
| `normal` | Только обычные тестовые вопросы (без маркера) |
| `open` | Только открытые вопросы (`> ℹ️ Открытый`) |
| `all` | Все вопросы: тестовые (базовые + обычные) + открытые |

### Параметры вывода

| Аргумент | По умолчанию | Описание |
|----------|-------------|----------|
| `-o`, `--output FILE` | `generated_test.md` | Имя выходного файла |
| `--title TITLE` | `Итоговый тест по ИБ` | Название итогового теста |
| `--show-source` | выкл. | Добавить под каждым вопросом пометку об источнике и оригинальном номере |

### Утилиты

| Аргумент | Описание |
|----------|----------|
| `--info` | Показать все найденные тесты и детальную статистику вопросов; тест не генерируется |
| `--seed N` | Фиксированный seed генератора случайных чисел. Одинаковый seed + одинаковые входные данные → всегда один и тот же тест |

## Примеры использования

### Посмотреть состав файла перед генерацией

```bash
python test_gen.py -i all_tests.md --info
```

Вывод:
```
📂 Загрузка файлов...
  ✅ all_tests.md  (3 тестов)  →  56 вопросов (23 базовых + 18 обычных + 15 открытых)
       • '15 Безопасность приложений': 21 вопр. (8 баз. + 7 обыч. + 6 откр.)
       • '14 Модель угроз': 20 вопр. (9 баз. + 8 обыч. + 3 откр.)
       • '13 Нормативные основы': 15 вопр. (6 баз. + 3 обыч. + 6 откр.)

📊 Итоговая статистика базы:
   Файлов:             1
   Тестов:             3
   Всего вопросов:     56
   ├─ Базовых (тест.): 23
   ├─ Обычных (тест.): 18
   └─ Открытых:        15
```

---

### Обычный тест из тестовых вопросов (по умолчанию)

```bash
python test_gen.py -i all_tests.md -n 20 --shuffle
```

Берёт 20 тестовых вопросов (базовых + обычных), перемешивает варианты ответов.

---

### Тест только из базовых вопросов

```bash
python test_gen.py -i all_tests.md -n 10 --type basic --shuffle
```

---

### Тест только из открытых вопросов

```bash
python test_gen.py -i all_tests.md -n 6 --type open
```

Варианты ответов не перемешиваются (их нет). В блоке ответов — развёрнутые тексты.

---

### Смешанный тест: тестовые + открытые вопросы

```bash
python test_gen.py -i all_tests.md -n 15 --type all --shuffle \
  --title "Итоговый экзамен по безопасности приложений" \
  -o exam.md
```

---

### Несколько файлов с весами

```bash
python test_gen.py -i all_tests.md extra.md -w 70 30 -n 20 --shuffle
```

70% вопросов из `all_tests.md`, 30% из `extra.md`.
Если `all_tests.md` содержит 3 теста, его вес `70` делится: по `23.3` на каждый тест.

---

### Воспроизводимый тест (одинаковый результат при повторном запуске)

```bash
python test_gen.py -i all_tests.md -n 20 --shuffle --seed 42
```

---

### Полный пример со всеми параметрами

```bash
python test_gen.py \
  -i all_tests.md extra_tests.md \
  -w 60 40 \
  -n 25 \
  --type all \
  --shuffle \
  --title "Итоговый экзамен по ИБ" \
  -o exam_final.md \
  --seed 2025
```

## Формат выходного файла

### Тест с тестовыми вопросами

```markdown
# Итоговый тест по ИБ

**Формат:** в каждом вопросе один правильный ответ.

---

>ℹ️ Базовый
### 1. Текст базового вопроса?
A. Вариант  
B. Вариант  
C. Вариант  
D. Вариант  

---

### 2. Текст обычного вопроса?
A. Вариант  
B. Вариант  
C. Вариант  
D. Вариант  

---

## ОТВЕТЫ — Итоговый тест по ИБ

**Тестовые вопросы:**
1-C, 2-A
```

### Тест с открытыми вопросами

```markdown
>✏️ Открытый
### 3. Что такое SQL-инъекция?

*Ответ:* _______________________________________________

---

## ОТВЕТЫ — Итоговый тест по ИБ

**Открытые вопросы:**

**3.** SQL-инъекция — уязвимость, при которой злоумышленник внедряет...
```

### Смешанный тест (тестовые + открытые)

Блок ответов содержит оба подраздела:

```markdown
## ОТВЕТЫ — Итоговый тест по ИБ

**Тестовые вопросы:**
1-C, 2-B, 4-A, 5-D

**Открытые вопросы:**

**3.** SQL-инъекция — уязвимость, при которой...

**6.** XSS — уязвимость веб-приложений, позволяющая...
```

## Логика весов при нескольких тестах в файле

Веса задаются **на файл**, а не на отдельный тест внутри него:

```
Файл A (вес 60) содержит 3 теста  →  каждый тест получает вес 60 / 3 = 20
Файл B (вес 40) содержит 1 тест   →  тест получает вес 40
```

Итого 4 теста с весами 20 : 20 : 20 : 40.
При запросе 20 вопросов из каждого теста файла A придёт ~4, из теста файла B — ~8.

## Обработка ошибок и граничные случаи

| Ситуация | Поведение |
|----------|-----------|
| Запрошено больше вопросов, чем есть в базе | Предупреждение; тест генерируется с максимально доступным количеством |
| Тестовый вопрос без буквенного ответа в блоке ответов | Предупреждение в stderr; вопрос пропускается |
| Открытый вопрос без текстового ответа | Предупреждение в stderr; вопрос включается в тест с пометкой «ответ не указан» |
| Тест без вопросов | Предупреждение; блок пропускается |
| Файл не найден | Ошибка с указанием пути; выход |
| Количество весов ≠ количеству файлов | Ошибка с пояснением |
| Нет вопросов нужного типа | Ошибка с пояснением |
| Кириллические буквы в блоке ответов (А, В, С, Д) | Автоматически нормализуются в латинские (A, B, C, D) |
